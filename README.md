# ToyProject

- 개발 환경: Visual Studio 2022(v143), C++20

## 기능

| 기능 | 사용 방법 | 구현 내용 |
| --- | --- | --- |
| 도형 생성 | 메뉴 생성에서 종류 선택 후 드래그 | 드래그 시작점과 현재 마우스 위치로 임시 도형을 표시, 마우스를 놓으면 실제 도형으로 등록 |
| 단일 선택 | 선택 모드에서 도형 클릭 | 겹친 도형이 있을 때 시각적으로 강조 |
| 영역 선택 | 빈 화면에서 드래그 | 점선 선택 상자를 표시, 놓는 시점에 선택 상자와 겹치는 모든 도형을 선택 |
| 선택 취소 | `Esc` | 선택 상태와 편집 상태를 해제 |
| 이동 | 도형 선택 후 `M`, 드래그 | 마우스 이동 동안 상대 오프셋으로 미리 보이고, 마우스를 놓을 때 실제 좌표에 이동량을 반영 |
| 편집 | 도형 선택 후 `E`, 드래그 | 도형 유형별 `Resize` 구현으로 크기를 조절, 점은 크기 조절 대상이 아님 |
| 선택 삭제 | 메뉴 삭제 > 선택삭제 | 선택된 도형의 고유 ID를 기준으로 목록에서 제거 |
| 전체 삭제 | 메뉴 삭제 > 전체삭제 | 선택 상태를 정리한 뒤 모든 도형을 제거 |
| 종료 | 메뉴 파일 > 끝내기 또는 창 닫기 | `WM_QUIT`를 전달해 메시지 루프를 종료 |
| 실행 상태 표시 | 창 제목 표시줄 | 매 프레임 경과 시간으로 계산한 FPS를 제목에 표시 |

도형이 생성될 때는 검정·빨강·초록·파랑 펜 및 브러시 조합 중 하나가 무작위로 적용

### 구분
생성 중 : 도형은 빨간 점선

선택 도형 : 굵은 빨간 선

편집 중 : 굵은 초록 선

## 설계

### 1. WinApp
프로그램의 메인 진입점

윈도우 등록, 메인 루프, 매니저 초기화/종료 담당

### 2. InputManager

입력을 담당하는 매니저 클래스

윈도우 메시지와 메뉴/키 명령을 unordered_map의 함수 테이블에 등록해 분기 ( LookupTable 방식 )

명령 추가 시 등록 함수와 처리 함수만 추가하면 되므로 확장을 고려해서 제작

### 3. ObjectManager

도형을 관리하는 매니저 클래스

Idle, Create, Selecting, Move, Edit 상태를 사용

마우스 다운, 마우스 이동, 마우스 업에 따라 이벤트에서 현재 상태에 따라 동작을 결정

모두 드래그라는 동일한 입력을 사용하기 때문에 State패턴을 활용, 해당 상태에선 하나의 동작만 수행하도록 제한

### 4. GDIManager

윈도우의 GDI를 관리하는 매니저 클래스

초기화 단계에서 HPEN, HBRUSH를 생성, 사용할 때 해당 오브젝트를 가져와 사용하는 방식을 채택함으로 오브젝트의 누수를 막음

더블버퍼링을 이용해 스크린티어링 현상을 방지

백버퍼를 생성하고 BitBlt을 하는 방식으로 WM_PAINT에서 렌더링을 하는게 아닌 렌더링 루프에서 렌더링되도록 변경

### 5. TimerManager

앱의 시간을 관리하는 클래스

해당 클래스를 통해 앱의 시간과 프레임을 표시, 매 프레임 DeltaTime을 계산하도록 설계

### 6. IShape

도형의 기본 클래스

ObjectManager에서 도형을 관리할 수 있도록 Interface클래스를 제작

### 7. 도형별 겹침 판정

EngineUtil::IsOverlap은 두 가지 도형에 맞는 기하 알고리즘을 구현

사각형 경계, 선분 교차, 점-선분 최단 거리, 원 중심과 경계의 최단 거리, 삼각형 내부 판정을 조합

선과 점은 PICK_TOLERANCE( 6px )를 적용해 매우 얇은 도형도 마우스로 선택할 수 있게 구현

## 동작 흐름

```text
Win32 메시지 입력
        ↓
InputManager ( 메시지, 커맨드 테이블에서 처리 )
        ↓
ObjectManager ( 현재 상태에 따른 생성/선택/이동/편집 )
        ↓
IShape 구현체 + EngineUtil ( 도형 좌표와 기하 판정 )
        ↓
GDIManager 백 버퍼 → BitBlt → 창 화면
```
